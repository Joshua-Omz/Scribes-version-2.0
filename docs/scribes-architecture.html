Key Data Flows
① OFFLINE DRAFT → PUBLISH
User types Note
→
Drift (SQLite) write
→
Reconnect detected
→
GET /sync?seq=N
→
Server returns Δ records
→
Merge into local DB
→
POST /posts → publish
Server sequence is authoritative — client clock is never trusted for ordering
② POST REVISION FLOW
Author edits Post
→
PATCH /posts/:id/revise
→
Snapshot current version → post_versions
→
Write new content to posts
→
Readers see latest version
→
Version History panel available
③ COLD-START FEED RANKING
New user selects categories
→
GET /feed/explore?cat=[]
→
Feed Handler queries posts + reactions + comments
→
Score = Σ reactions + comments (90d window)
→
Return ranked list
No implicit signals — view counts and read times are never stored
④ NOTIFICATION DELIVERY
Interaction event fires
→
Handler enqueues job
→
@mention / admin?
→
YES → real-time push
/
NO → batch queue → digest schedule
