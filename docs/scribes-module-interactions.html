Module Interaction Map

// semi-low-level abstract · how modules call each other · data contracts

Client Modules (Flutter)
🗂️ SyncModule client/sync
  • startSync()Entry point. Reads last server_sequence from Drift. Calls API.
  • applyDelta()Merges server records into local SQLite. Increments local sequence.
  • resolveConflict()Last-write-wins: server timestamp authoritative in v1.
  • pushPending()Sends locally-created drafts/notes that haven't been uploaded yet.
✏️ EditorModule client/editor
  • initEditor()Loads WYSIWYG. Accepts Note or Draft as seed content.
  • onChange()Debounced local Drift write via StorageModule. No API call.
  • onPublish()Validates categories + scripture refs, then calls PostModule.create().
  • onRevise()Calls PostModule.revise(). Triggers version snapshot on backend.
💾 StorageModule client/storage (Drift)
  • write(entity)Upsert to SQLite. Tags record with local_only=true if offline.
  • read(query)Typesafe Drift query. Returns stream for Riverpod to watch.
  • getSequence()Returns last known server_sequence for sync checkpoint.
  • purge(ids)Hard-deletes local records confirmed deleted by server.
📰 FeedModule client/feed
  • loadFeed()Calls GET /feed. Checks if user is cold-start (no follows). Riverpod AsyncNotifier.
  • loadExplore()Calls GET /explore?categories=[]. Paginated.
  • renderPost()Emits Post widget. Links to PostDetailModule for tap.
🔔 NotifModule client/notifications
  • listen()Polls GET /notifications or receives push for real-time events.
  • onMention()Real-time path. Directly surfaces in-app banner. No batching.
  • processBatch()Digest events arrive on schedule. Grouped by type and rendered.
🌐 APIClient client/api
  • get/post/patch()Base HTTP methods. Attaches JWT. Handles 401 refresh.
  • onConnected()Network watcher. Fires SyncModule.startSync() on reconnect.
  • serialize()Converts Dart models to JSON request bodies.
  • deserialize()Maps JSON responses to typed Dart model classes.
Backend Handler Modules (Go)
📝 PostHandler api/post
  • CreatePost()Validates author, categories, scripture refs. Writes post + assigns server_sequence. Enqueues follower feed invalidation.
  • RevisePost()Snapshots current content → post_versions. Writes new content. Bumps current_version. Returns updated Post.
  • GetVersionHistory()Queries post_versions by post_id. Returns ordered list.
  • SoftDeletePost()Sets is_deleted=true. Post becomes tombstone. No hard delete.
  • ExportPost()Calls ExportService. Returns .md or .txt byte stream.
🔄 SyncHandler api/sync
  • GetDelta()Accepts last_seq from client. Queries notes, drafts, posts WHERE server_sequence > last_seq AND owner=caller. Returns delta payload.
  • PushPending()Accepts bulk upsert of local-only records. Assigns server_sequence to each. Returns ack with server sequences.
  • assignSequence()Internal. Uses DB-level sequence (nextval). Never uses client timestamp.
📡 FeedHandler api/feed
  • GetFollowingFeed()Queries posts WHERE author_id IN (user's follows). Reverse-chron. Paginated by cursor.
  • GetExploreFeed()Queries posts with interaction score = SUM(reactions + comments) in 90d. No view counts.
  • GetColdStartFeed()Called when follow count = 0. Passes user's onboarding categories to GetExploreFeed(). Same score formula.
💬 DMHandler api/messages
  • SendMessage()Checks mutual follow OR approved request. Text-only enforced. Writes to messages table.
  • RequestConversation()Creates pending MessageRequest. Notifies recipient. No message sent until approved.
  • BlockConversation()Sets blocked=true on conversation. Hides thread from both parties. Prevents further sends.
  • DeleteMessage()Soft-delete: body replaced with "[Message deleted]". Sender only.
🛡️ AuthMiddleware api/middleware
  • ValidateJWT()Runs on every request. Extracts user_id + role. Injects into context.
  • RequireRole()Guard for admin-only routes. Returns 403 if role != SUPER_ADMIN.
  • RateLimit()Per-IP sliding window. Applied before handler execution.
⚡ NotifWorker worker/notifications
  • enqueue(event)Called by handlers. Writes to in-memory job channel (buffered goroutine).
  • dispatch()Background goroutine. Reads channel. Routes to realtime or batch path.
  • flushBatch()Runs on schedule. Groups batched notifications. Sends digest.
  • sendRealtime()@mention / admin alert. Immediate delivery. Does not wait for digest.
Critical Interaction Sequences
SEQ-01 User publishes a Post while online
EditorModule → EditorModule onPublish() validates categories + scripture refs client-side
EditorModule → APIClient post(POST /posts, body: {content, caption, categories, scripture_refs})
APIClient → AuthMiddleware ValidateJWT() — injects user_id into request context
AuthMiddleware → PostHandler CreatePost(ctx, req) — writes post row, assigns server_sequence via nextval()
PostHandler → NotifWorker enqueue(PostCreated) — async; does not block response
PostHandler → APIClient Returns 201 {post, server_sequence}
APIClient → StorageModule write(post) — updates local Drift cache; sets local_only=false
SEQ-02 App reconnects after offline drafting
APIClient → SyncModule onConnected() fires → SyncModule.startSync()
SyncModule → StorageModule getSequence() → returns last known server_sequence N
SyncModule → SyncHandler GET /sync?seq=N — server returns all records with sequence > N
SyncModule → StorageModule applyDelta(records) — upserts each record; updates local sequence to server max
SyncModule → SyncHandler POST /sync/push — uploads local_only=true records; server assigns sequences
SyncModule → StorageModule write(serverAck) — stamps records with server_sequence; clears local_only flag
SEQ-03 Author revises a published Post
EditorModule → APIClient onRevise() → PATCH /posts/:id/revise {newContent: JSONB}
AuthMiddleware → PostHandler RevisePost(ctx) — verifies caller == post.author_id
PostHandler → PostHandler INSERT post_versions (snapshot current content, version_number, snapshotted_at)
PostHandler → PostHandler UPDATE posts SET content=newContent, current_version++, server_sequence=nextval()
PostHandler → APIClient Returns 200 {post: updatedPost, versionCount: N}
SEQ-04 @mention triggers real-time notification
APIClient → CommentHandler POST /posts/:id/comments {body: "...@handle..."}
CommentHandler → CommentHandler parseMentions(body) → resolves @handle to user_ids
CommentHandler → NotifWorker enqueue({type: MENTION, recipients: [user_ids], ref: commentId})
NotifWorker → NotifWorker dispatch() reads channel: type == MENTION → sendRealtime() path
NotifWorker → NotifModule Push delivery → client NotifModule.onMention() → in-app banner
Cross-Module Call Reference
WHO CALLS WHOM Caller → Callee · method · contract
APIClient → SyncModule startSync() on reconnect event
SyncModule → StorageModule getSequence() · applyDelta() · write() primary consumer of StorageModule
EditorModule → StorageModule write(draft/note) debounced; offline-safe
EditorModule → APIClient post(POST /posts) · patch(PATCH /posts/:id/revise) publish & revise paths
FeedModule → APIClient get(GET /feed | /explore) paginated cursor
AuthMiddleware → All Handlers injects ctx.user_id, ctx.role runs before every handler
PostHandler → NotifWorker enqueue(PostCreated) non-blocking goroutine channel write
SyncHandler → PostgreSQL sqlc: SELECT * WHERE server_sequence > $1 AND user_id=$2 server_sequence never from client
NotifWorker → NotifModule push(MENTION) immediately · flushBatch() on schedule two delivery paths
